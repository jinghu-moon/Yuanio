### 🟡 问题2: 生产环境日志泄露 (P1)

**修复状态**: ✅ **已修复核心模块**

**修复内容**:

1. **安装结构化日志库**
   - 使用 pino 替代 console 调用
   - 支持环境变量控制日志级别

2. **创建日志模块** (`crates/relay-server/src/logger.ts`)
   - 自动脱敏敏感字段: token, password, deviceId, sessionId, fcmToken
   - 结构化日志输出

3. **替换关键敏感日志**
   - `jwt.ts`: 移除密钥相关 console 调用
   - `index.ts`: 替换包含 deviceId 的 3 处关键日志
     - FCM token 注册日志
     - 设备连接日志
     - 设备断开日志

**验证数据**:
- 修复前: 412 处 console 调用
- 已修复: 核心敏感信息泄露点
- 剩余: 非敏感日志可后续迁移
