# WS 重连与异常矩阵（最小手工集）

目标：在 Socket.IO 下线前，验证原生 WS 的断线恢复、会话连续性与 ACK 可靠性。

| ID | 场景 | 操作 | 期望 |
| --- | --- | --- | --- |
| T1 | 网络抖动 | 客户端断网 5s → 恢复 | 自动重连成功，队列/ACK 恢复 |
| T2 | Relay 重启 | 运行中重启 relay | WS 断开后重连，`hello` 重新握手 |
| T3 | 进程崩溃 | Relay 进程被杀 | 客户端进入重连，恢复后重新加入会话 |
| T4 | 半开连接 | 断开 TCP（非正常 close） | 超时检测触发重连（无卡死） |
| T5 | 协议不兼容 | `protocolVersion` 主版本不一致 | 收到 `error` 帧并关闭 |
| T6 | 前后台切换 | App 后台 30s → 前台 | 自动恢复连接与会话状态 |

备注：
- 执行顺序建议：T1 → T2 → T3 → T4 → T5 → T6。
- 记录：失败时保存日志与时间点，便于复现。
